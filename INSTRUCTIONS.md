# BookShelf - Книжный магазин с CI/CD

## Описание проекта

**BookShelf** — учебное веб-приложение (книжный магазин), демонстрирующее полный CI/CD-пайплайн:
- Фронтенд на React + Vite
- Автоматическое тестирование через GitHub Actions
- Автоматический деплой в папку `server/` только при успешных тестах

## Структура

```
├── src/                       # Исходный код React-приложения
│   ├── components/            # UI-компоненты (Header, BookCard, Cart и др.)
│   ├── data/books.js          # Моковые данные книг
│   ├── utils/cartUtils.js     # Логика корзины
│   └── __tests__/             # Тесты (Vitest)
├── server/                    # Папка-"сервер" (обновляется ТОЛЬКО через CI/CD)
├── .github/workflows/ci-cd.yml  # Конфигурация GitHub Actions
├── package.json
└── vite.config.js
```

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск в режиме разработки

```bash
npm run dev
```

Откройте http://localhost:5173 в браузере.

### 3. Запуск тестов

```bash
npm test
```

Или в режиме наблюдения:

```bash
npm run test:watch
```

### 4. Сборка для продакшна

```bash
npm run build
```

Собранные файлы окажутся в папке `dist/`.

## Настройка GitHub

### 1. Создайте репозиторий на GitHub

Зайдите на https://github.com/new и создайте новый репозиторий.

### 2. Инициализируйте git и запушьте

```bash
git init
git add .
git commit -m "Initial commit: BookShelf bookstore with CI/CD"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
git push -u origin main
```

### 3. Настройте права для GitHub Actions

Чтобы CI/CD мог пушить обновления в `server/`, нужно дать Actions разрешение на запись:

1. Зайдите в Settings вашего репозитория
2. В меню слева: **Actions** → **General**
3. Прокрутите до секции **Workflow permissions**
4. Выберите **Read and write permissions**
5. Нажмите **Save**

## Как работает CI/CD

### Схема работы

```
Разработчик          GitHub Actions          Папка server/
    │                      │                      │
    │  git push            │                      │
    ├─────────────────────►│                      │
    │                      │  npm install          │
    │                      │  npm test             │
    │                      │  npm run build        │
    │                      │                      │
    │                      │  Тесты прошли? ──────►│ Копирует dist/ → server/
    │                      │  Тесты упали? ───✕    │ НЕ обновляет server/
    │                      │                      │
```

### Что происходит при пуше:

1. **Вы пушите код** в ветку `main`
2. **GitHub Actions** автоматически запускает пайплайн (файл `.github/workflows/ci-cd.yml`)
3. Пайплайн выполняет:
   - `npm ci` — установка зависимостей
   - `npm test` — запуск всех тестов
   - `npm run build` — сборка проекта
4. **Если все тесты прошли:**
   - Собранные файлы из `dist/` копируются в `server/`
   - Изменения коммитятся и пушатся обратно в репозиторий
5. **Если тесты упали:**
   - Папка `server/` остаётся без изменений
   - На GitHub появляется красный крестик у коммита

### Ключевой момент

Папка `server/` **обновляется ТОЛЬКО при успешном прохождении тестов**. Это эмулирует реальный процесс деплоя на продакшн-сервер: сломанный код никогда не попадёт к пользователям.

## Проверка работы CI/CD

### Успешный деплой

1. Внесите любое изменение в код (например, измените заголовок в `src/components/Footer.jsx`)
2. Закоммитьте и запушьте:
   ```bash
   git add .
   git commit -m "Update footer text"
   git push
   ```
3. Перейдите на GitHub → вкладка **Actions**
4. Дождитесь зелёной галочки
5. Проверьте папку `server/` — она обновилась

### Провал тестов

1. Специально сломайте тест — например, в `src/data/books.js` удалите одну из книг
2. Закоммитьте и запушьте
3. На GitHub → **Actions** появится красный крестик
4. Папка `server/` **не обновилась** — старая рабочая версия осталась
